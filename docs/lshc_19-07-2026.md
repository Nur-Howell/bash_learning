# Linux System Health Checker - Version 1.1 Changes

## Overview
Version **1.1** focuses on improving the readability and usability of the system health report while keeping the original functionality of report generation intact.

---

## Changes Made

### Improved Output Formatting
- Added section headers to organize the report:
  - `===== Storage =====`
  - `===== Memory =====`
  - `===== Services =====`
- Added spacing between sections to improve readability.

---

### Refined Disk Usage Reporting
**Version 1.0**
- Displayed the complete `df -h` output.

**Version 1.1**
- Extracts only the root (`/`) partition usage.
- Uses `awk` to isolate the required column.
- Uses `sed` to remove the `%` symbol before processing.
- Displays a simplified output:

```text
Usage: 47%
```

---

### Refined Memory Reporting
**Version 1.0**
- Displayed the complete `free -h` output.

**Version 1.1**
- Extracts only the most relevant memory information:
  - Total Memory
  - Used Memory
  - Available Memory
- Uses `awk` to filter the required fields.
- Uses `sed` to remove the `Gi` suffix for cleaner output.

Example:

```text
Total: 7.6
Used: 2.1
Available: 5.5
```

---

### Service Status Checks
**Version 1.0**
- Printed the raw output of:

```bash
systemctl is-active apache2
systemctl is-active ssh
```

**Version 1.1**
- Stores the service status in variables.
- Uses conditional (`if`) statements to determine whether each service is running.
- Displays user-friendly status messages.

Example:

```text
apache2 is running...
ssh is not running...
```

---

### Color-Coded Service Status
Added ANSI color variables:

- Green → Running
- Red → Not Running

This allows service health to be identified immediately without reading every line.

---

### Variable Usage
Introduced variables to store processed command output, including:

- `disk_usage`
- `memory_total`
- `memory_used`
- `memory_available`
- `apache_act`
- `ssh_act`

This improves readability and makes future modifications easier.

---

### Report Generation
The report generation process remains unchanged.

The script still:
- Creates the `logs/` directory if it does not exist.
- Generates timestamped report filenames.
- Displays the report on-screen.
- Saves the report using `tee`.

---

## Summary

Version **1.1** improves the script by focusing on:

- Cleaner output formatting
- Easier-to-read system information
- Simplified disk and memory reporting
- Human-readable service status
- Color-coded service health
- Better use of variables while maintaining the original report generation functionality
